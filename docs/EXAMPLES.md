# Examples

Reference workflows demonstrating common patterns. All examples live in
`tests/fixtures/` or are generated by `minion init`.

## `minion init` templates

```bash
minion init my-workflow --template <blank|fix-issue|code-review|security-audit>
```

| Template         | Demonstrates                                                    |
|------------------|-----------------------------------------------------------------|
| `blank`          | Minimal shell — one `cmd` step                                  |
| `fix-issue`      | Agent plan → implement → test → commit loop with gates          |
| `code-review`    | `map` over changed files, per-file review, post summary comment |
| `security-audit` | Multi-stage audit with parallel scanners and a final report     |

## Pattern catalog

### 1. Sequential cmd steps

```yaml
name: basic-pipeline
steps:
  - name: fetch
    type: cmd
    run: "curl -s https://api.example.com/data"
  - name: process
    type: cmd
    run: "echo '{{ steps.fetch.stdout }}' | jq '.count'"
    output_type: integer
```

### 2. Agent with planning + execution

```yaml
name: fix-issue
steps:
  - name: plan
    type: agent
    prompt: "Plan the fix for issue {{ target }}"
  - name: implement
    type: agent
    prompt: "Execute this plan:\n{{ steps.plan.response }}"
    # session: shared (default) — forks from plan's session
```

### 3. Gate-controlled retry

```yaml
scopes:
  attempt:
    steps:
      - name: fix
        type: agent
        prompt: "Fix failing tests. Previous attempt: {{ scope.value }}"
      - name: test
        type: cmd
        run: "cargo test"
      - name: check
        type: gate
        condition: "{{ steps.test.exit_code == 0 }}"
        on_pass: break
        on_fail: continue
    outputs: "{{ steps.fix.response }}"

steps:
  - name: fix_loop
    type: repeat
    scope: attempt
    max_iterations: 5
    initial_value: "initial"
```

### 4. Map over files

```yaml
name: review-pr
scopes:
  review:
    steps:
      - name: view
        type: cmd
        run: "git diff origin/main -- {{ scope.value }}"
      - name: analyze
        type: agent
        prompt: "Review this diff:\n{{ steps.view.stdout }}"
    outputs: "{{ steps.analyze.response }}"

steps:
  - name: files
    type: cmd
    run: "git diff --name-only origin/main"
    output_type: lines
  - name: per_file
    type: map
    items: "{{ steps.files.output | join(sep=',') }}"
    scope: review
    parallel: 4
```

See `tests/fixtures/map_serial.yaml` for the serial variant.

### 5. Parallel independent checks

```yaml
name: ci-checks
steps:
  - name: quality
    type: parallel
    steps:
      - name: lint
        type: cmd
        run: "cargo clippy --all -- -D warnings"
      - name: test
        type: cmd
        run: "cargo test"
      - name: fmt
        type: cmd
        run: "cargo fmt --check"
```

### 6. Prompt templates (external files)

```yaml
name: stack-aware-lint
prompts_dir: prompts

steps:
  - name: lint_prompt
    type: template                       # reads prompts/lint_prompt.md.tera
  - name: fix
    type: agent
    prompt: "{{ steps.lint_prompt.rendered }}"
```

`prompts/lint_prompt.md.tera`:

```markdown
You are a linter for {{ stack.name }}.
Target: {{ target }}
Tools: lint={{ stack.tools.lint }}, test={{ stack.tools.test }}
```

See `tests/fixtures/template_step.yaml` for a full example.

### 7. Stack-aware dynamic prompt

```yaml
steps:
  - name: stack_prompt
    type: template
    prompt: "fix-lint/{{ stack.name }}"   # → prompts/fix-lint/react.md.tera
```

The registry at `prompts/registry.yaml` defines which stack is detected from
the repository (e.g. presence of `package.json` + `react` dep → stack `react`).

### 8. Config layering

```yaml
name: mixed-models
config:
  global:
    timeout: 300s
  agent:
    model: claude-opus-4-6
  patterns:
    "lint.*":
      model: claude-haiku-4-5

steps:
  - name: lint_check          # matches "lint.*" pattern → haiku
    type: agent
    prompt: "Find style issues in {{ target }}"
  - name: deep_review         # no pattern match → opus from agent layer
    type: agent
    prompt: "Do a thorough review of {{ target }}"
  - name: urgent_review
    type: agent
    prompt: "Quick sanity check of {{ target }}"
    config:
      model: claude-haiku-4-5 # step-inline wins even over pattern
```

## Running the examples

```bash
# With sandbox (default)
minion execute code-review.yaml -- 42

# Host mode
minion execute --no-sandbox basic-pipeline.yaml

# Dry-run (prints tree, resolves config, no execution)
minion execute --dry-run fix-issue.yaml -- 123

# Resume after interruption
minion execute --resume implement fix-issue.yaml -- 123

# JSON output (for pipelines)
minion execute --json code-review.yaml -- 42 | jq '.total_cost_usd'
```
